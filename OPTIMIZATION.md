### 1.1 RAG 进出题链路（回答"知识库怎么接"）
现状：retrieveChunks 只用于首页建议；出题纯靠模型参数，知识库与题目零关联。
- generate-questions 增加 `chunks` 入参：TrainPage 出题前按维度检索 top-3 注入 prompt，要求"题目必须考察 chunks 中的知识点，输出 source 字段标注依据"
- 知识库扩容：8 条 → 200+ 条（训练营课件切片 + 真实面经题解 + AI 产品案例），按 dimension_tags/keywords 结构入 course_chunks
- 前端题目下方展示"考点来源"折叠条 → 可解释性 = 产品亮点 + 面试演示点

### 1.2 评分校准（回答"准确率怎么评"= LLM-as-judge 实操）
现状：单 LLM 按通用锚点打分，无维度 rubric、无一致性验证。
- 每维度写评分 rubric（如 narrative 维度：结构 40 / 量化 30 / tradeoff 30）注入 get-feedback prompt
- 建 20-30 条校准集（题目+典型作答+人工分），每次改评分 prompt 跑一致性（|LLM分-人工分|≤10 的占比 ≥80% 才上线）
- 校准脚本可复用主线项目的 golden-set-runner 思路
- **产出即简历素材**："在自研上线产品中建立 LLM-as-judge 评分校准机制，一致率 X%"

### 1.3 题库混合策略（回答"题库怎么做"）
纯生成 → 生成+精选混合：人工审核过的真题池（面经来源）占每轮 2 题、LLM 生成 3 题；生成题被用户标"题目有问题"两次即下架进待审池——题库质量的数据飞轮。
